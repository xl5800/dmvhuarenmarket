# DMV 华人集市上线发布说明

## 你需要的线上组件

第一版真正上线，建议用这套：

- Supabase：登录、数据库、图片存储、审核数据
- Vercel：部署网站
- Namecheap / Cloudflare：购买和解析域名

## 第 1 步：创建 Supabase 项目

1. 打开 Supabase，创建新项目。
2. 进入 SQL Editor。
3. 复制 `supabase-schema.sql` 的全部内容并运行。
4. 先用你自己的邮箱注册一次。
5. 回到 SQL Editor，运行：

```sql
update public.profiles
set role = 'admin'
where email = '你的邮箱@example.com';
```

这样你就是后台管理员，可以审核帖子。

## 第 2 步：拿到项目配置

在 Supabase 后台进入：

Project Settings -> API

复制：

- Project URL
- anon public key

然后把 `supabase-config.example.js` 复制成：

```text
supabase-config.js
```

填入：

```js
window.DMV_SUPABASE_CONFIG = {
  url: "你的 Supabase Project URL",
  anonKey: "你的 Supabase anon public key",
};
```

注意：不要把 service role key 放到前端。

## 第 3 步：前端要改的地方

现在网页还是本地演示版，数据存在浏览器里。要真正发布，需要把这些动作改成 Supabase：

- 登录：从本地假登录改成 Supabase Auth 邮箱登录
- 发布帖子：从 `localStorage` 改成 insert 到 `public.listings`
- 读取列表：从 `localStorage` 改成 select `status = approved`
- 我的发布：读取当前登录用户自己的 listings
- 审核台：管理员读取 `status = pending` 的 listings
- 通过 / 拒绝 / 下架：管理员 update listing status
- 举报：insert 到 `public.reports`

## 第 4 步：图片上传

第一版可以先继续用图片链接。真正让用户上传图片时：

1. 在 Supabase Storage 创建 bucket：`listing-images`
2. 限制上传大小，比如每张 5MB。
3. 前端上传图片后，把返回的公开 URL 存到 `listings.image_url`。

## 第 5 步：部署网站

最快方式：

1. 把 `dmv-huaren-market` 文件夹上传到 GitHub。
2. Vercel 选择这个 GitHub 仓库。
3. Framework 选 Other / Static。
4. 部署后绑定域名。

## 第 6 步：上线前必须打开的保护

- Supabase Auth 开启邮箱确认。
- Supabase RLS 保持开启。
- 只允许用户创建 `pending` 状态帖子。
- 联系方式登录后可见。
- 管理员审核通过后才公开显示。
- 每条帖子保留举报入口。
- 给 Supabase 设置用量提醒。

## 推荐上线顺序

1. 先上线 DMV 租房发布、审核、展示。
2. 再上线二手发布。
3. 再上线图片上传。
4. 最后再接真实地址解析、地图搜索和距离筛选。

早期最重要的是：真实用户能发布，管理员能审核，前台能展示。
